System status
  • Healthy Latest poll succeeded and every device responded.
  • Setup needed No Bluetooth connection configured yet. Visit Settings → Setup and click "Find my dongle".
  • BLE not connected Your Bluetooth connection is configured but the dongle isn't reachable. Check the BT-2 is powered + in range; on Docker, check the passthrough.
  • Comms slow No successful poll in 2+ minutes. Usually a brief BLE wobble; clears on the next cycle.
  • X errors Last poll completed but some devices didn't reply. Common after a daemon restart (BlueZ holds the BT-2 connection for ~60s).
  • Stale No successful poll for 5+ minutes. Check Bluetooth power / range to the dongle.
  • Offline Scheduler isn't running. Visit Settings → System → Restart daemon.

Solar-aware charger pause · Pro

Pause your AC charger when solar is covering the load. The daemon re-enables it before the bank drops too low. Cooldown + a hard SoC floor protect against weather forecast errors. Off by default. Manual toggles always win; the rule won't override an output you just turned on or off yourself. The recommended control point is a Shelly Plug S Gen2 or Tasmota plug on the mains lead going into your AC charger. Add one under smart_plugs: in config.yaml; the dropdown picks it up automatically.

Backup & restore

Download a single archive containing the full SQLite database (all history, devices, alerts) plus config.yaml and your local-UI password hash. Restore on a fresh install to come back exactly where you were — useful before swapping the SD card, moving to new hardware, or running a risky firmware update.

Download backup
Restore:

Restore is destructive. It overwrites whatever you ticked above, then restarts the daemon. The previous config.yaml is kept as config.yaml.restored.bak in case you need to back out. Tip: untick Configuration to bring your history back onto a fresh, clean config.

Discovery telemetry

Anonymous: when you scan for BLE / Modbus devices, the appliance sends a fingerprint of anything our drivers don't recognise to wattpost.cloud, so we know which hardware to support next. Off by default. No full MAC, no IP, no account info. Just OUI, advertised name, and a few bytes of manufacturer data.

Anonymous install ping

On the daily update check, the appliance can include an anonymous install ID + version so we can count how many installs exist and which release they run. Off by default. No account, no location, no device data. The update check runs either way; this only controls whether the anonymous ID rides along.

Reset to defaults

Wipes all configured transports, devices, exporters, alerts and smart-scene rules so this appliance behaves like a freshly installed one. Your SQLite history, web password and (by default) cloud pairing are kept. A timestamped config.yaml.bak is written first so you can restore from disk if you change your mind.
